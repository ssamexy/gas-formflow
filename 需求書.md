# GAS FormFlow 需求規格書

## 專案名稱

```text
gas-formflow
```

## 專案標語

用 Google Forms + Google Apps Script 打造私有化、免費、AI-ready 的表單工作流。

使用者只需要設定一次，之後即可在手機上打開自己架設的 Web App，搭配 AI 產生 JSON，一鍵建立 Google Form、Google Sheet 回應表、統計摘要與通訊軟體公告文案。

---

## 專案定位

GAS FormFlow 是一個自架型 Google Apps Script Web App。

它的目標是讓個人、教會、社團、小型非營利組織，可以不用架伺服器、不用付費 SaaS，就能快速完成：

```text
AI 討論需求
→ JSON spec
→ Google Form 建立
→ Google Sheet 回應表建立
→ LINE / 通訊軟體公告
→ QR code
→ 基本統計整理
→ 統計摘要輸出
```

v1 不內建 LLM API，使用者可先透過 ChatGPT / Gemini / Claude 等外部 AI 討論需求並產生 JSON。

v2 可擴充 LLM API，讓自然語言需求、JSON 產生、表單設計建議、統計摘要、公告文案生成都整合到同一個 Web App。

## 使用者分級

### 小白使用者

特色：

* 不會使用 CLI
* 不會使用 clasp
* 只想照步驟複製貼上
* 主要用手機操作表單產生流程

安裝方式：

```text
打開自己的 Google Apps Script 後台
→ 貼上本 repo 提供的 Code.gs
→ 貼上本 repo 提供的 Index.html
→ 執行一次授權
→ 部署為 Web App
→ 之後都用自己的 Web App URL 操作
```

### 進階使用者

特色：

* 懂 CLI
* 會使用 GitHub
* 可能使用 Codex / Claude Code / Gemini CLI / 其他 AI agent
* 想用 clasp 部署與維護

安裝方式：

```text
clone repo
→ 安裝 clasp
→ 登入 Google 帳號
→ 建立或綁定 Apps Script 專案
→ clasp push
→ 部署 Web App
```

README 需同時提供小白版與進階版部署方法。

## 設計原則

### 免費

v1 只使用：

```text
Google Apps Script
Google Forms
Google Sheets
Google Drive
```

不依賴付費 SaaS。

### 私有化

使用者自行部署自己的 Apps Script Web App。

產生的 Google Form 與 Google Sheet 都建立在使用者自己的 Google 帳號底下。

README 必須提醒：

```text
不要直接使用陌生人部署好的 Web App URL。
若使用別人部署的 Web App，表單與資料可能會建立在對方 Google 帳號底下。
自行部署後，表單與資料會留在自己的 Google 帳號底下。
```

### 手機友善

主要操作需能在手機瀏覽器完成：

```text
貼 JSON
驗證
預覽
建立表單
複製連結
複製公告文案
查看 QR code
```

### 小白可用

repo 必須提供：

```text
dist/Code.gs
dist/Index.html
```

讓使用者只需要複製兩個檔案到 Google Apps Script。

### 進階可維護

repo 也需提供多檔案開發版：

```text
src/
```

方便工程使用者維護與擴充。

### AI-ready

v1 不內建 LLM API，但 JSON schema 要適合 AI 產生。

v2 可加入 LLM API。

原則：

```text
數字統計由 GAS 計算。
LLM 只負責需求理解、JSON 生成、文字整理、語意摘要。
```

## 小白部署流程：繁體中文

README 與 `docs/install.zh-TW.md` 需提供以下步驟。

```text
1. 打開 Google Apps Script。
2. 建立新專案。
3. 將專案名稱改成 GAS FormFlow。
4. 打開 dist/Code.gs。
5. 複製全部內容。
6. 回到 Apps Script，貼到 Code.gs。
7. 新增 HTML 檔案，命名為 Index。
8. 打開 dist/Index.html。
9. 複製全部內容。
10. 貼到 Apps Script 的 Index.html。
11. 儲存專案。
12. 在 Apps Script 上方選擇 setup 或 doGet 測試函式。
13. 執行一次，完成 Google 授權。
14. 點選「部署」→「新增部署作業」。
15. 類型選擇「網頁應用程式」。
16. 執行身分選擇「我」。
17. 存取權依需求選擇：
    - 只有我自己
    - 擁有連結的任何人
18. 完成部署後，複製 Web App URL。
19. 之後手機打開這個 URL，即可貼 JSON 產生表單。
```

需提醒：

```text
第一次授權時，Google 可能會顯示未驗證應用程式。
因為這是使用者自己建立的 Apps Script 專案。
請確認程式碼來自自己信任的 repo 後再繼續授權。
```

## Web App v1 功能需求

Web App 需提供以下功能：

### 1. JSON 輸入

大型 textarea，讓使用者貼上 AI 產生的 JSON。

### 2. 載入範例

提供按鈕載入內建範例 JSON。

### 3. 驗證 JSON

檢查：

```text
JSON 格式是否正確
schemaVersion 是否支援
title 是否存在
items 是否存在
item type 是否支援
選項題是否有 options
grid 題是否有 rows / columns
key 是否合理
```

### 4. 表單預覽

建立前顯示：

```text
表單標題
表單說明
題目數量
每題題型
每題標題
每題選項
是否必填
```

### 5. Sheet 結構預覽

建立前顯示預計建立的 Sheet 分頁：

```text
Form Responses 1
Clean_Data
Question_Meta
Summary
Announcement
Generator_Log
```

並預覽：

```text
Clean_Data 欄位
可統計欄位
不適合自動統計的欄位
統計摘要規劃
```

### 6. 建立 Google Form

根據 JSON 建立 Google Form。

### 7. 建立 Google Sheet

建立回應用 Google Sheet，並將 Google Form response destination 指向該 Sheet。

### 8. 產生公告文案

根據 `lineMessageTemplate` 產生可複製文案。

支援變數：

```text
{title}
{publishedUrl}
{editUrl}
{sheetUrl}
{deadlineText}
{notice}
{qrCodeUrl}
```

### 9. 產生 QR code

v1 盡量支援表單填寫連結的 QR code。

優先方案：

```text
前端使用內嵌 QR code JavaScript library，在 Index.html 直接產生 QR code。
```

原則：

```text
優先不依賴外部 QR code API。
避免使用已不穩定或已淘汰的 Google Image Charts QR API。
若要使用外部 QR API，需做成可選 provider，不可作為唯一方案。
```

v1 成功後需顯示：

```text
表單填寫連結
表單 QR code
可下載 QR code 圖片，若實作成本合理
可複製 QR code 圖片，若瀏覽器支援
```

若 v1 實作 QR code 成本超出預期，則至少保留：

```text
QR code placeholder
QR code provider 設計
README 標示此功能為 v1.1
```

### 10. 顯示結果

成功後顯示：

```text
表單填寫連結
表單編輯連結
Google Sheet 連結
表單 QR code
公告文案
建立時間
```

### 11. 錯誤處理

錯誤訊息需適合非工程使用者閱讀。

不要只顯示 stack trace。

## JSON Schema v1

JSON 根層建議格式：

```json
{
  "schemaVersion": "1.0",
  "title": "表單標題",
  "description": "表單說明",
  "notice": "重要提醒",
  "deadlineText": "截止時間",
  "confirmationMessage": "送出後訊息",
  "sheetName": "回應試算表名稱",
  "lineMessageTemplate": "公告文案模板",
  "analysis": {
    "enabled": true,
    "primaryKey": "name",
    "generateCleanData": true,
    "generateSummary": true,
    "summaryFields": []
  },
  "items": []
}
```

---

## item 格式

每個題目建議包含：

```json
{
  "key": "stable_field_key",
  "type": "multipleChoice",
  "title": "題目文字",
  "helpText": "補充說明",
  "required": true,
  "options": [],
  "analysis": {
    "role": "group",
    "summary": "countOptions"
  }
}
```

`key` 是穩定欄位名稱，用於：

```text
Clean_Data 欄位
Question_Meta
Summary
後續統計
v2 LLM 摘要
```

## Google Sheet 輸出需求

建立表單時，同時建立 Google Sheet。

Sheet 需包含以下分頁：

```text
Form Responses 1
Clean_Data
Question_Meta
Summary
Announcement
Generator_Log
```

### Form Responses 1

Google Forms 原始回應表。

### Clean_Data

依照 item key 建立穩定欄位。

例如：

```text
timestamp | name | gender | area | attendance_intention
```

### Question_Meta

記錄題目 metadata：

```text
key | title | type | required | options | analysis_role | summary_type
```

### Summary

根據 metadata 產生基本統計。

v1 支援：

```text
回覆總數
單選題選項統計
複選題選項統計
下拉選單選項統計
線性量表平均與分布
日期統計
時間統計
數字欄位加總 / 平均
缺漏檢查
疑似重複檢查
```

### Announcement

放置產生後可複製的公告文案。

### Generator_Log

記錄每次建立表單：

```text
createdAt | title | publishedUrl | editUrl | sheetUrl
```

## 統計分析原則

v1 不用 LLM 自動理解表單意圖。

系統根據 JSON 中的：

```text
type
key
options
analysis
```

做可預測的結構化統計。

### GAS 負責

```text
人數
比例
加總
平均
選項統計
缺漏檢查
重複檢查
名單整理
```

### LLM 未來可負責

```text
自然語言需求轉 JSON
JSON 修復
表單設計建議
開放式回答摘要
統計結果轉公告
給不同對象的摘要版本
```

## LLM API v2 規劃

v2 可加入 Bring Your Own API Key。

API key 儲存在使用者自己的 Apps Script Properties。

LLM 功能可包含：

```text
自然語言需求 → JSON spec
JSON 修復與補完
表單設計建議
提醒不利統計的題型
統計結果轉 LINE 公告
開放式回答摘要
給同工 / 主管 / 群組的不同版本摘要
```
